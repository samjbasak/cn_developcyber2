Unit 4: Securing and Defending Online Systems

Extended Project.

Role: Cyber Engineer.

Brief 1 - Research a working cyber security system

Unit 4 - Brief Phase 1: -

Create of a Research Document based on your exploration of network systems:

1. Begin investigation of a WAMP server and identify the main components

2. Explanation of the main components of the system

3. Description of how the components fit together

4. Discuss notes with teammates in standups

5. Highlight key objectives of the new system yet to be designed

Brief 2 - Plan to build cyber safe website or server

Unit 4 - Brief Phase 2: -

Creation of a Technical Specification document that details the following:

1. Skeleton plan of the system including diagrams, goals (what kind of website it is etc)

2. Set outcomes including safety features

3. Explanation of hardware requirements

4. Explanation of software aspects of system

Brief 3 - Develop a cyber safe website or server

Unit 4 - Brief Phase 3: -

To create your system, you need to perform these tasks:

1. Detail some notes about why the specification chosen is correct. Do this as you build.

2. Consider how a website functions when you’re building it and how possible breaches could occur and why. Create notes as you go.

3. Explanation of system requirements and how they’re keeping this particular system safe

4. Configure the system in a robust manner

Brief 4 - Test the system against common threats.

Unit 4 - Brief Phase 4: -

To test your system effectively, you need to perform these tasks:

1. Develop a test plan

2. Explanation of the purpose of test procedures.

3. Provide detail about the expected test results

4. Describe actual test results and what they mean

5. Make adjustments to the system in light of testing

Brief 5 - Evaluate the effectiveness of the system.

Unit 5 - Brief Phase 5: -

To evaluate the system, a project retrospective will take place between you and your team. In this you will discuss the following:

1. Analyse results against original objectives.

2. Evaluation of system and its purpose – what would we keep/remove?.

3. Justify design decisions against objectives.

4. Explore possible improvements based on usage.

5. Determine effectiveness of the system.

6. Recommendations for improvements to the system for future-proofing.
